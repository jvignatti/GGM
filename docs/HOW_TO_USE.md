# How to Use the Dashboard
## Step-by-Step Guide for Regular Users

---

## What You Need

Before you start, make sure you have:

1. **`dashboard.html`** — the dashboard file (from this repository or shared by your team)
2. **`grants_data.json`** — the data file (from your authorized internal source — **never share this file publicly**)
3. A modern web browser (Chrome, Edge, or Safari recommended)

That's it. No software to install. No login. No internet connection needed.

---

## Step 1 — Open the Dashboard

Double-click `dashboard.html`. It will open in your default web browser.

You will see a clean screen with an upload area in the center. The dashboard is empty at this point — it contains no data until you load the JSON file.

> **Tip:** If the file opens as plain text instead of a webpage, right-click the file → Open With → your browser.

---

## Step 2 — Load Your Data

You have two options:

**Option A — Drag and Drop (easiest)**
Drag the `grants_data.json` file from your desktop or folder directly onto the upload area in the dashboard. Drop it there.

**Option B — Click to Browse**
Click the upload area. A file browser window opens. Navigate to where `grants_data.json` is saved and select it.

**What happens next:**
The dashboard loads instantly. All tabs populate with data. The data timestamp in the top bar shows when the JSON was last generated.

> **Privacy note:** Your data never leaves your computer. The dashboard runs entirely in your browser. Nothing is uploaded to any server.

---

## Step 3 — Navigate the Dashboard

Use the tab bar across the top to move between sections:

| Tab | Go here to... |
|---|---|
| **Overview** | See the big picture — total grants, money, risk summary |
| **Grants** | Browse and filter all grants |
| **Invoices** | Check invoice status across grants |
| **Quarterly Reports** | Check QTR status across grants |
| **Financial** | See budget vs. spend for each grant |
| **Supervisors** | See who is assigned to what |
| **Signals** | See which grants need attention right now |

---

## Step 4 — Filter and Search

On the **Grants** tab and most other tabs, you can:

- **Search** by typing in the search box (grant number, agency name, etc.)
- **Filter** using the dropdown menus (grant type, supervisor, priority level, signal type)
- **Sort** by clicking any column header — click again to reverse the sort
- **Combine filters** — all filters work together

To reset all filters, click the **Clear Filters** button.

---

## Step 5 — Read the Priority Score

Every grant has a **Priority Score** from 0 to 100.

| Score | Color | Meaning |
|---|---|---|
| 75–100 | 🔴 Red | Critical — needs immediate attention |
| 50–74 | 🟠 Orange | High — review this week |
| 25–49 | 🟡 Yellow | Medium — monitor closely |
| 0–24 | 🟢 Green | Low — on track |

The score is calculated from financial risk, compliance risk, and timeline risk. See [`MATH.md`](MATH.md) for the full formula.

---

## Step 6 — Read the Signals

The **Signals** tab shows every active alert across the portfolio.

Each signal means something specific:

| Signal | Plain English |
|---|---|
| Invoice Late | An invoice hasn't been submitted in over 60 days |
| QTR Late | A quarterly report hasn't been submitted in over 60 days |
| Low Utilization | The grant is spending much slower than expected |
| High Utilization | The grant is nearly out of budget |
| Over Budget | Spending has exceeded the award amount |
| Near End Date | The grant ends within 60 days |
| No Activity | No invoices or QTRs at all — grant may be stalled |
| Missing Supervisor | No supervisor is assigned |

A grant can have multiple signals at once. Sort by signal count to find the most critical grants first.

---

## Step 7 — Export a View

To save what you're looking at:

1. Apply whatever filters you want
2. Click the **Export CSV** button
3. A CSV file downloads to your computer
4. Open it in Excel

The export always reflects the current filtered view — not the full dataset.

---

## How to Refresh the Data

The dashboard is a snapshot. It does not auto-update.

To get current data:
1. Ask the person who runs the pipeline to re-run it
2. Obtain the new `grants_data.json`
3. Open the dashboard again and load the new file

Check the data timestamp in the top bar to know how current your data is.

---

## Frequently Asked Questions

**Q: Can I save my filters for next time?**
A: Not currently. The dashboard resets when you close the browser. Re-apply your filters when you reload.

**Q: Can I open the dashboard on my phone?**
A: Yes, but the experience is optimized for desktop. Tables are scrollable on mobile.

**Q: What if the dashboard looks blank after loading the JSON?**
A: Check that you loaded the correct file (`grants_data.json` generated by the pipeline). If the file is empty or corrupted, the dashboard will show no data.

**Q: Can I share the dashboard HTML file?**
A: Yes — the HTML file contains no data and is safe to share. Never share the JSON file outside your organization.

**Q: What if the data looks wrong?**
A: The dashboard reflects exactly what is in the JSON file. If data looks incorrect, the source may need to be re-exported and the pipeline re-run.
